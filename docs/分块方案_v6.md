# 电动自行车说明书 · 分块方案 v6（终版）

> 融合 RAGFlow 学习成果 + v5 优化 + 语义分块 + 故障三元组，适用于维修类说明书知识库构建。

---

## 一、设计目标

将一本电动自行车说明书切分为**可独立检索、语义自洽、链路可追溯**的最小知识单元。每个 chunk 不仅是一段文本，而是一个携带了"它是什么类型""它回答什么问题""它涉及哪些部件""故障场景下症状-原因-动作是什么"的完整知识块。

---

## 二、语义类型体系

不再按"text/table/warning"这种格式类型分类，而是按**内容表达的语义**分类：

| 语义类型 | 定义 | 说明书中的典型内容 | 检索场景 |
|----------|------|-------------------|----------|
| **操作步骤** | 按顺序执行的动作序列 | 充电步骤、安装步骤、维护步骤 | "电池怎么充电""后视镜怎么装" |
| **故障诊断** | 症状→原因→处理方法的结构化知识 | PART 9 故障排除表、常见故障判断表 | "控制器故障灯亮了""充不进电" |
| **风险警告** | 违反会产生人身/财产损害的安全须知 | 三级警告（危险/警告/注意） | "充电有什么安全注意事项" |
| **参数查询** | 精确数值型信息 | 技术参数表（电压/功率/车速）、三包期限 | "最高时速多少""控制器保修多久" |
| **部件说明** | 部件的空间位置、功能、关联关系 | VLM 增强的部件标注图描述、仪表指示灯 | "前照灯在哪""READY灯是什么意思" |
| **电路拓扑** | 电气连接关系、信号线路、保护器件 | VLM 增强的电气原理图描述 | "刹车断电经过哪些节点" |
| **概述说明** | 不具备上述特征的前言/免责/总述 | 前言、免责声明、目录 | 辅助上下文，不作为独立检索目标 |

### 语义类型判定规则

```
逐 chunk 判定：

if 文本包含 "步骤" 或 连续 3 个以上 ^\d+\.  的有序列表:
    → 操作步骤

elif 文本包含 故障/异常/排除/原因/处理/维修方案:
    → 故障诊断

elif 文本以 > ⛔/⚠/ℹ 开头或含 危险/警告/注意 标签:
    → 风险警告

elif 文本是 Markdown 表格 且 含数值型数据:
    if 含 保修/三包/质保: → 参数查询（保修子类）
    else: → 参数查询

elif 文本含 VLM 增强内容:
    if 含 电池→控制器→电机 等连接关系: → 电路拓扑
    else: → 部件说明

else:
    → 概述说明
```

## 三、分块流水线（6 个 Phase）

```
电动自行车说明书_for_chunking.md
    │
    ▼
Phase 1: 结构解析
    │  识别 PART/H2/H3 层级 + 表格/警告/VLM/有序步骤 边界
    │  产出：带层级的节点树
    ▼
Phase 2: 原子块保护
    │  表格/警告/有序步骤/VLM块 标记为不可切分
    │  故障码/参数对 通过 children_delimiters 拆为子块
    ▼
Phase 3: 语义分类
    │  对每个块判定语义类型（操作步骤/故障诊断/风险警告/...）
    ▼
Phase 4: 上下文挂载
    │  表格 ← 前一段文字挂载
    │  故障块 ← 所属章节标题挂载
    │  VLM 块 ← 父 H2 标题挂载
    ▼
Phase 5: 大小控制 + 父子关系
    │  512t 目标，768t 上限，50t 碎片合并
    │  overlap: 正常 0t，强制拆分 50t
    │  建立 parent_id / child_ids / mom_id
    ▼
Phase 6: LLM 后处理 + 故障三元组提取
    │  关键词 + 预设问题 + 领域标签
    │  故障块：提取 symptom→cause→action 三元组
    ▼
chunks_v6.json
```

## 四、核心规则

### 4.1 任务驱动切分

当文本段包含以下关键词时，**强制提升为独立 chunk**（不被合并到相邻段落）：

| 触发词 | 说明 | 示例 |
|--------|------|------|
| `检查` | 检查类操作 | "请在骑行前按照以下方式仔细检查您的车辆" |
| `故障` | 故障描述 | "故障现象与排除" |
| `排除` | 故障排除 | "常见故障及维修方案" |
| `维修` | 维修方法 | "联系授权服务站检测维修" |
| `处理` | 处理方法 | "处理方法：重新插拔插头" |
| `更换` | 更换操作 | "更换保险丝" |

**实现**：
```python
TASK_TRIGGERS = ["检查", "故障", "排除", "维修", "处理", "更换"]

def is_task_driven(text: str) -> bool:
    return any(trigger in text[:50] for trigger in TASK_TRIGGERS)
```

### 4.2 故障三元组提取

针对语义类型为"故障诊断"的 chunk，提取结构化三元组：

```json
{
  "semantic_type": "故障诊断",
  "fault_triplet": {
    "symptom": "接通电源，仪表正常显示，调速失灵",
    "component": "控制器",
    "cause": "控制器或电机故障",
    "action": "联系授权服务站检测维修",
    "action_type": "送修",
    "can_diy": false
  }
}
```

**提取方式**：

```python
# 方式 A：从结构化表格中直接提取（PART 9 故障表已是三列）
# 故障现象 | 故障原因及排除方法
# → symptom = 故障现象列
# → cause + action 从"故障原因及排除方法"列中拆分

# 方式 B：LLM 辅助提取（非表格文本）
prompt = """这段文本描述了一个故障场景。请提取：
1. 症状(symptom)：发生了什么问题
2. 部件(component)：涉及的车辆部件
3. 原因(cause)：为什么发生
4. 处理方法(action)：怎么解决
5. 是否可自助(can_diy)：用户自己能处理吗？true/false

文本：{text}

输出JSON：{"symptom":"...", "component":"...", "cause":"...", "action":"...", "can_diy":true/false}"""
```

**本说明书中可提取三元组的来源**：

| 来源 | 页面 | 自然格式 | 三元组数量 |
|------|:---:|------|:---:|
| 故障现象与排除 | P25 | 表格：故障现象 → 故障原因及排除方法 | ~10 条 |
| 常见故障及维修方案 | P26 | 表格：问题现象 → 原因 → 处理方法 | ~10 条 |
| 充电器故障判断 | P26 | 表格：问题现象 → 原因 → 处理方法 | ~5 条 |
| 电池故障判断 | P26 | 表格：问题现象 → 原因 → 处理方法 | ~5 条 |
| 电气原理图 VLM 描述 | P32 | VLM 提取的故障指示灯说明 | ~5 条 |

### 4.3 Chunk+Query 对齐元数据

每个 chunk 携带以下字段，供检索层做精确评分：

```json
{
  "query_alignment": {
    "component_name": ["控制器", "电机"],
    "fault_type": "调速失灵",
    "action_type": "送修",
    "part_number": "PART 9",
    "is_safety_critical": false
  }
}
```

| 字段 | 来源 | 检索用途 |
|------|------|----------|
| `component_name` | 部件提取或 LLM 标注 | 精确匹配"控制器""电池"等部件名 |
| `fault_type` | 故障三元组的 symptom | 精准召回同症状的故障 |
| `action_type` | LLM 分类（自修/送修/检查/更换） | "自己能修吗"类问题优先命中自修项 |
| `part_number` | 所属 PART | 限定搜索范围 |
| `is_safety_critical` | 风险警告 chunk 标记 | 安全类查询优先召回 |

## 五、不同类型 chunk 的处理策略

| 语义类型 | 大小控制 | 父块聚合 | children 保护 | 三元组 | LLM 后处理 |
|----------|:---:|:---:|:---:|:---:|:---:|
| 操作步骤 | 512t 目标 | ✅ | 步骤行 children 拆分 | — | keywords + questions |
| 故障诊断 | 完整表格行 | ✅ | 每行 children | ✅ symptom→cause→action | questions |
| 风险警告 | 原子块 | ✅ | — | — | — |
| 参数查询 | 原子块 | ✅ | 参数对 children | — | — |
| 部件说明 | 随父 section | ✅ | — | — | keywords |
| 电路拓扑 | 随父 section | ✅ | — | 故障指示灯→含义 | keywords |
| 概述说明 | 512t 目标 | 作为父 chunk | — | — | — |

## 七、新增字段总览

| 字段 | 层级 | 来源 | 用途 |
|------|------|------|------|
| `semantic_type` | chunk 顶层 | Phase 3 规则判定 | 检索时按类型过滤/加权 |
| `fault_triplet` | chunk 顶层 | Phase 6 LLM 提取 | 故障场景精确召回 |
| `fault_triplet.symptom` | triplet 内 | 表格列或 LLM | 匹配用户描述的故障现象 |
| `fault_triplet.component` | triplet 内 | LLM 提取 | 部件级精确过滤 |
| `fault_triplet.cause` | triplet 内 | 表格列或 LLM | 提供诊断信息 |
| `fault_triplet.action` | triplet 内 | 表格列或 LLM | 提供修复方案 |
| `fault_triplet.action_type` | triplet 内 | LLM 分类 | 自修/送修/检查/更换 |
| `fault_triplet.can_diy` | triplet 内 | LLM 判断 | 二分类过滤 |
| `query_alignment.component_name` | alignment 内 | LLM 提取 | 部件名精确匹配 |
| `query_alignment.fault_type` | alignment 内 | symptom 映射 | 故障类型召回 |
| `query_alignment.action_type` | alignment 内 | action_type 映射 | 操作类型过滤 |
| `query_alignment.is_safety_critical` | alignment 内 | warning_level 判定 | 安全查询加权 |
